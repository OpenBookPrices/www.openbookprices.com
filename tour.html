API Tour

This is a quick tour of the OpenBookPrices.com API, how to use it and what it returns.

RESTful

All requests to the API are GET requests using http.

There is no authentication or API key required.

JSON

The response data is formatted using JSON.

JSONP

If you add a callback query parameter to the url the response will be in JSONP so you call the API from a browser without worrying about the same origin restrictions.

(We'd suggest not using JSONP if you can help it as the responses won't be edge cached.)

CORS

Access-Control-Allow-Origin: *

The API also sends the above header so that CORS can be used (if the browser supports it).

(If you can use CORS you should, it means you'll get faster responses from the edge caches on the CDN)

Caching

The responses are intended to be cached and the correct headers are set and timestamps and ttls are in the returned data.

The cache will not hold stale data, please don't try to cache-bust as you will not get anything fresher. We try to limit the traffic that our fetchers send to the vendors sites and caching the results is part of that.
