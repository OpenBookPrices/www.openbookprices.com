Examples

These are simple examples that you should be able to try in your browser. The API returns JSON which your browser will display more nicely with a plugin like JSONView or a tool like Postman (for Chrome).

For more see the full API docs.

Basic book details

http://api.openbookprices.com/v1/books/9780131103627/details

Get some very basic details about the book (eg title, authors).

Prices

http://api.openbookprices.com/v1/books/9780131103627/prices

No country or currency specified so will geolocate using your IP address and redirect. Then displays all vendors that ship to where you are and the details that we currently hove cached. Does not block on scraping prices so returns very fast.

http://api.openbookprices.com/v1/books/9780131103627/prices/FR/EUR

As above but with the country and currency specified (FR for France and EUR for Euro). If you leave out the currency then the primary currency for the country will be looked up and redirected to.

The returned JSON contains the country and currency so you don't need to detect the redirect and parse the final url.

http://api.openbookprices.com/v1/books/9780131103627/prices/FR/EUR/amazon_us

Get the pricing for an individual vendor, in this case amazon_us with shipping to Fronce and the amounts converted into Euros. If we don't have fresh data then we'll fetch it from the vendor using their API or scraping them so this might take longer to return, but once it does it is cached on the CDN so it'll be faster next time.

Note that the response is identical to the entries in the results array for the non-vendor request, so you can use the same piece of code to process them both.
