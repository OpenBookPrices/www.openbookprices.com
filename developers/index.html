We are developers and we've suffered plenty of terrible APIs. Weird naming, crazy auth, lousy documentation, overly complicated, expensive. Blech!

Our API is none of these because we've built it all with a view to being developer friendly:

  • There is no auth, or signup, or tokens.
  • It's free (both speech and beer).
  • It only returns JSON.
  • The URL is the request (no headers, no sessions).
  • Auto-corrects arguments using redirects when possible.
  • Response is easy to work with.
  • Built in geolocation based on requestor's IP address.
  • Heavily cached using global CDN.

Quick start

The API should be fairly self explanatory. Most endpoints have links to related ones and the urls are hackable. Try it by going to http://api.openbookprices.com/ (you might need to install a JSON viewer).

There are also lots of examples with notes which give a pretty good overview of the whole API.
